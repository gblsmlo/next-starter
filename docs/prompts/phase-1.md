Role: You are a senior frontend architect specializing in Next.js App Router and Feature-Based Architecture (FBA).

Instructions: Initialize a production-ready Next.js project in the current directory using pnpm, configured for FBA with TypeScript path aliases.

Constraints (non-negotiable):
- Package manager: pnpm only — never npm or yarn
- Versions: always @latest — never pin version numbers
- Path aliases: explicit FBA aliases only — never "@/*"
- On failure: stop immediately, report exact error, suggest fix, do not continue

Steps:
1. Run the Next.js scaffolding command:
   pnpm create next-app@latest . --typescript --tailwind --eslint --app --src-dir --no-import-alias --yes
   
   Then add environment and compiler dependencies:
   pnpm add @t3-oss/env-nextjs zod next react react-dom
   pnpm add -D @next/env babel-plugin-react-compiler

2. Replace tsconfig.json "paths" with FBA aliases:
   "@features/*" → "./src/features/*"
   "@components/*" → "./src/components/*"
   "@routes/*" → "./src/routes/*"
   "@types/*" → "./src/types/*"
   "@lib/*" → "./src/lib/*"
   "@hooks/*" → "./src/hooks/*"
   Also set "baseUrl": "." and "target": "ES2022"

3. Update next.config.* to include reactStrictMode: true

4. Preserve any pre-existing docs/ and .agent/ directories without modification

End goal: A working Next.js App Router project where all FBA path aliases resolve correctly and `pnpm dev` starts without errors.

Narrowing: Do not add any extra dependencies, plugins, or abstractions beyond what is specified. Do not create feature directories — only configure the aliases. Verify each file exists before marking the step complete.

Output format:
- Framework initialized (name + version)
- Confirmation of all 6 FBA aliases configured
- List of files created or modified
- Final status: "✅ Ready for Phase 2: Biome Configuration"
```
